# التقرير الفني المرجعي — مشروع إعادة هيكلة شبكة NetLink على CCR2004

## 1) الهدف التنفيذي
- توحيد خدمات الشبكة في CCR2004 (الوحش الأبيض) لخدمتي PPPoE وHotspot.
- إلغاء الاعتماد على جهازي الإدارة/الهوتسبوت القديمين بعد نجاح الترحيل.
- تنفيذ تصميم جديد من الصفر بدون ترحيل عشوائي لقواعد قديمة.

## 2) حدود المشروع
- لا تعديل على Silver-Bridge.
- Proxmox يبقى على 192.168.99.x كما هو.
- التطبيق يبدأ من بعد Silver-Bridge فقط.

## 3) البيانات المؤكدة
- Silver-Bridge: 192.168.99.1
- CCR2004 الجديد: 192.168.99.200
- SAS4 Radius: 192.168.99.10
- Shared Secret: 123456
- Broadcast/Devices Mgmt: 192.168.88.0/24
- Proxmox/Services: 192.168.99.0/24
- PPPoE Pool: 10.0.0.0/24
- Hotspot Pool: 10.20.30.0/24

## 4) الهيكلية الجديدة
- شبكة إدارة للبنية (88 و99).
- شبكة PPPoE منفصلة للمشتركين.
- شبكة Hotspot منفصلة للمشتركين.
- السماح بالإنترنت فقط عبر مصادقة PPPoE أو Hotspot.

## 5) معالجة الثغرة القديمة
- الثغرة السابقة: تعيين IP يدوي من 88/99 ثم الحصول على إنترنت.
- المعالجة الجديدة:
  - اعتبار 88/99 شبكات إدارة فقط.
  - منع NAT مباشر من 88/99 إلى الإنترنت.
  - سياسة Filter تمنع المرور غير المصرح.
  - السماح فقط لحركة المشتركين الموثقين.

## 6) مراحل التنفيذ
1. تجهيز CCR2004 (هوية، وقت، DNS، IP إدارة).
2. بناء Bridge/VLAN بشكل نظيف.
3. تشغيل PPPoE وربطه مع Radius.
4. تشغيل Hotspot وربطه حسب التصميم.
5. تطبيق سياسات أمان ومنع bypass.
6. اختبار فعلي ثم تحويل الكوابل ثم إلغاء الأجهزة القديمة.

## 7) معايير النجاح
- PPPoE يعمل بثبات.
- Hotspot يعمل بثبات.
- لا يوجد إنترنت بدون مصادقة.
- لا تأثر على Silver-Bridge أو Proxmox.

## 8) خطة الرجوع
- إبقاء الأجهزة القديمة جاهزة أثناء الانتقال.
- عند فشل اختبار حرج يتم الرجوع فورًا للوضع السابق.

## 9) ملاحظات تشغيل
- يمنع إدخال FastTrack عشوائي قبل التحقق من تعارضه مع PPPoE/Hotspot.
- أي تعديل لاحق يكتب في هذا الملف كمرجع تشغيلي.
