# تقرير تنفيذ الهوت سبوت الكامل — NetLink Core CCR2004

## مكان حفظ الملفات على جهازك (Windows)
- المجلد الرئيسي محفوظ هنا:
  - `C:\Users\aljabareen\Desktop\snp9\NetLink Enterprise DB`
- المسار يعني: **سطح المكتب** > **snp9** > **NetLink Enterprise DB**

## هدف التقرير
توثيق خطوات نشر الهوت سبوت بشكل كامل على CCR2004، مع تسجيل المشاكل التي ظهرت أثناء التنفيذ، الأسباب الجذرية، والإجراءات التصحيحية حتى الوصول إلى تشغيل ناجح ومستقر.

## بيئة العمل
- اسم الراوتر: `NetLink-Core-CCR2004`
- خادم RADIUS: `192.168.99.10`
- شبكة الهوت سبوت: `10.20.30.0/24`
- Gateway: `10.20.30.1`
- DHCP Pool: `10.20.30.2-10.20.30.254`

## خطوات تنفيذ الهوت سبوت (التسلسل الصحيح)
1. تجهيز واجهة المشتركين (Bridge/VLAN) حسب تصميم الشبكة الفعلي.
2. إسناد عنوان الشبكة للواجهة الصحيحة:
   - `10.20.30.1/24`
3. إنشاء أو التأكد من وجود Pool باسم `hs-pool` بالنطاق:
   - `10.20.30.2-10.20.30.254`
4. تشغيل Hotspot Setup بالقيم الأساسية:
   - SSL Certificate: none
   - SMTP Server: `0.0.0.0`
   - DNS Servers: `1.1.1.1`, `8.8.8.8`
   - DNS Name: `netlink.wifi`
5. إنشاء قاعدة NAT لخروج مستخدمي الهوت سبوت:
   - `src-address=10.20.30.0/24`
   - `action=masquerade`
   - `out-interface=ether1`
6. مراجعة Server Profile وLogin Profile لمنع الدخول التلقائي غير المطلوب.

## المشاكل التي ظهرت أثناء التنفيذ والحلول

### المشكلة 1: الهاتف لا يحصل على IP
**العرض:** رسالة "تعذر الحصول على عنوان IP" عند الاتصال.

**السبب الجذري:** عدم تطابق واجهة الهوت سبوت مع مسار المرور الفعلي (الربط كان على bridge بينما الإعداد على VLAN أخرى).

**الإصلاح:**
- نقل IP `10.20.30.1/24` إلى `br-subscribers`.
- ربط DHCP Server على `br-subscribers`.
- تعديل `hotspot1` للعمل على `br-subscribers`.

---

### المشكلة 2: Hotspot Server يظهر باللون الأحمر
**العرض:** `hotspot1` غير فعّال (أحمر).

**السبب الجذري:** عدم تطابق بين Interface وAddress Pool أو غياب عنوان الشبكة على نفس الواجهة.

**الإصلاح:**
- ضبط `Interface = br-subscribers`.
- ضبط `Address Pool = hs-pool`.
- التأكد من وجود `10.20.30.1/24` على نفس الواجهة.

---

### المشكلة 3: DHCP Server يظهر باللون الأحمر
**العرض:** إدخال DHCP غير فعّال (أحمر).

**السبب الجذري:** حذف غير مقصود في DHCP Networks أو عدم تطابق نطاق الشبكة مع إعداد الخادم.

**الإصلاح:**
- حذف إدخال DHCP المعطّل.
- إعادة تنفيذ `DHCP Setup` على الواجهة الصحيحة.
- التأكد من ظهور Network scope الصحيح في تبويب Networks.

---

### المشكلة 4: دخول تلقائي بدون صفحة تسجيل
**العرض:** العميل يتصل بالإنترنت مباشرة دون طلب اسم مستخدم/كلمة مرور.

**السبب الجذري:** بقاء آليات المصادقة التلقائية مفعّلة (Cookie / MAC Cookie / IP Binding).

**الإصلاح:**
- تعطيل `Cookie`, `MAC Cookie`, `MAC`, `Trial` من Login Profile.
- حذف الجلسات النشطة من Active.
- حذف Cookies المخزنة.
- مراجعة/تنظيف IP Bindings (خصوصًا bypass/regular غير المقصودة).

## نتيجة الاختبار النهائي
- صفحة الهوت سبوت ظهرت بنجاح.
- تسجيل الدخول تم بنجاح.
- العميل حصل على IP: `10.20.30.209`.
- الخدمة تعمل بشكل طبيعي ومستقر.

## توصيات تشغيلية
- تجنب التعليقات العربية داخل MikroTik إذا كانت تظهر كـ `???`، واستخدم English/Franco.
- قبل أي تغيير إنتاجي: خذ Backup وفعّل Safe Mode.
- نفّذ تغييرات الهوت سبوت كحزمة مترابطة: Interface + IP + DHCP + Hotspot + NAT.

## قائمة تحقق سريعة لأي عطل مستقبلي
1. هل `10.20.30.1/24` موجود على الواجهة الصحيحة؟
2. هل DHCP Server مربوط على نفس الواجهة؟
3. هل `hotspot1` يعمل على نفس الواجهة ونفس الـ Pool؟
4. هل NAT Rule الخاصة بالهوت سبوت موجودة ومفعّلة؟
5. هل Login Profile مضبوط حسب سياسة الدخول (مع/بدون auto-login)؟
6. هل توجد Cookies أو IP Bindings قديمة تؤثر على السلوك؟
