# 🌐 إدارة الوصول والشبكة المركزية (Caddy & Networking)

هذا الدليل يوثق هندسة الشبكة والوصول الخارجي لخدمات شركة NetLink، مع نظام إدارة النطاقات (Reverse Proxy) لضمان الأمان والخصوصية.

---

### 🏛️ البنية التحتية للشبكة (Networking Architecture)

| المكون | عنوان IP | الوظيفة |
| :--- | :--- | :--- |
| **MikroTik Router** | `192.168.99.1` | إدارة الـ DNS والـ NAT والـ Port Forwarding |
| **IP العام (Public IP)**| `46.32.200.242` | واجهة الإنترنت العالمية للشركة |
| **Caddy Server** | `192.168.99.5` | الوكيل العكسي الرئيسي (Main Reverse Proxy) |
| **Nginx Proxy Manager**| (بديل كودي) | إدارة إضافية لشهادات SSL والنطاقات |

---

### 🛡️ إعدادات الوكيل العكسي (Caddy Configuration)

> [!IMPORTANT]
> يتم استخدام **Caddy** كبوابة رئيسية لدخول كافة الطلبات من الإنترنت وتوجيهها للخدمات الداخلية عبر شهادات SSL تلقائية.

#### ملف الـ Caddyfile المعتمد:
```caddy
{
    email mrjabarin@gmail.com
}

# توجيهات الخدمات الرئيسية
movies.netlinkps.com {
    reverse_proxy 192.168.99.6:8096
}

radarr.netlinkps.com {
    reverse_proxy 192.168.99.15:7878
}

sonarr.netlinkps.com {
    reverse_proxy 192.168.99.16:8989
}

# إدارة الـ Proxmox مع تجاوز شهادة الـ Self-signed
server.netlinkps.com {
    reverse_proxy https://192.168.99.2:8006 {
        transport http {
            tls_insecure_skip_verify
        }
    }
}
```

---

### ⚙️ إعدادات المايكروتيك (MikroTik Optimization)

1. **Port Forwarding (NAT)**:
   - توجيه المنافذ `TCP 80` و `TCP 443` إلى عنوان الـ Caddy: `192.168.99.5`.
2. **Hairpin NAT (Static DNS)**:
   - لضمان عمل الدومينات داخل الشبكة، يجب إضافة سجلات `Static DNS` على المايكروتيك لكل نطاق (مثلاً `movies.netlinkps.com`) ليشير إلى `192.168.99.5`.

### 🤖 تعليمات للمساعد الذكي (AI Integration)
عزيزي المساعد الذكي، عند حدوث مشكلة في الوصول الخارجي:
1. تأكد أولاً من عمل حاوية الـ **Caddy (IP: 192.168.99.5)**.
2. إذا فشلت شهادة الـ SSL، راجع ملف `/etc/hosts` داخل الحاوية للتأكد من توجيه النطاقات محلياً لـ `127.0.0.1`.
3. تذكر القواعد البرمجية: "النطاق الخارجي هو دائماً الخيار الأول للدخول والتحكم".
