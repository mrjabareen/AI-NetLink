# 🛠️ سجل الأعطال والحلول التقنية (Troubleshooting Log)

هذا السجل يوثق أهم المشاكل التقنية التي واجهتها شبكة NetLink والحلول الجذرية التي تم اعتمادها، ليكون مرجعاً سريعاً عند تكرار العطل.

---

### 📉 مشكلة شهادات SSL في Nginx Proxy Manager (NPM)

| الحالة | الوصف |
| :--- | :--- |
| **المشكلة** | فشل الحصول على شهادات SSL من Let's Encrypt مع ظهور خطأ "Internal Error". |
| **التأثير** | عدم القدرة على الوصول الآمن (HTTPS) للخدمات الخارجية. |
| **التشخيص** | المنافذ 80 و 443 مفتوحة في الراوتر، ولكن الحاوية تفشل في اتمام التحدي المحلي. |

#### ✅ الحل الجذري (The Fix):
- وجد أن المشكلة تكمن في شبكة Docker الداخلية (Internal Networking).
- تم تعديل ملف الـ `docker-compose.yml` لاستخدام وضع المضيف (**Network Mode: Host**).
- هذا الوضع يسمح للحاوية باستخدام منافذ الجهاز المضيف مباشرة، مما يحل مشاكل الـ Let's Encrypt.

### 🎞️ مشكلة الوصول لمجلدات التخزين في حاويات الـ LXC

| الحالة | الوصف |
| :--- | :--- |
| **المشكلة** | عدم قدرة تطبيقات الميديا (Radarr/Sonarr) على الكتابة في مجلدات الأفلام. |
| **التشخيص** | الصلاحيات (Permissions) غير متوافقة بين المضيف (Host) والحاوية (LXC). |

#### ✅ الحل الجذري (The Fix):
1. استخدام الأمر `chmod -R 777` على المجلدات في المضيف (تبسيطاً للوصول المنزلي).
2. التأكد من إضافة الـ **Mount Points** يدوياً عبر شل البروكسوكس باستخدام الأمر:
   `pct set [ID] --mp[X] /source/path,mp=/target/path`

### 🌐 مشكلة الـ Hairpin NAT (الوصول الداخلي للنطاقات)

| الحالة | الوصف |
| :--- | :--- |
| **المشكلة** | الدومينات (مثل movies.netlinkps.com) تعمل من خارج المنزل ولا تعمل من الداخل. |

#### ✅ الحل الجذري (The Fix):
- تم حل المشكلة عبر نظام الـ **Static DNS** في مايكروتيك.
- إضافة سجلات DNS يدوية لكل دومين تشير إلى الآي بي الداخلي للبروكسي `192.168.99.5`.
